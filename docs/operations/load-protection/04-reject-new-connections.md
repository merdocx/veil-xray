# Отказ только новым подключениям (шаг 4, опционально)

Включайте после **базовой линии** и заполненной таблицы SLO, когда видны повторяющиеся авралы.

## Скрипт

[../../../scripts/load-protection/stress-mode-nft.sh](../../../scripts/load-protection/stress-mode-nft.sh) — включает/выключает правило nftables: **блокировать новый входящий TCP на порт** (по умолчанию 443), **не трогая** уже установленные соединения.

Переменные окружения см. в шапке скрипта. **Не** блокируйте `127.0.0.1:10085` (API Xray).

## Гистерезис

Порог «включить стресс» и «выключить» должны различаться, иначе правило будет **флапать**.

Пример логики внешнего таймера: при MemAvailable &lt; **A** включить; при MemAvailable &gt; **B** выключить, где **B &gt; A**.

## Осторожно с connlimit по IP

Несколько пользователей за одним NAT получат общий лимит.
