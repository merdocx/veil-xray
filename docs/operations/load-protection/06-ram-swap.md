# RAM и swap (шаг 6)

## Чеклист

1. Зафиксировать объём RAM и картину по графикам (шаг 2).
2. Проверить `vm.swappiness` (`sysctl vm.swappiness`) — на серверах часто 10–60; 100 даёт агрессивный swap.
3. При **стабильной** нехватке памяти под нормальный пик — **увеличить RAM** предпочтительнее длительного тюнинга.
4. Проверить соседние сервисы на том же хосте (БД без лимитов, раздувающиеся логи).
