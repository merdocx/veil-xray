## Предлагаемая таблица порогов

Действия ориентировочные: при «предупреждении» — смотреть Netdata и лог; при «критично» — готовить отказ новым сессиям / масштабирование по [04](04-reject-new-connections.md).

| Метрика | Как смотреть | Предупреждение | Критично | Действие |
|---------|----------------|----------------|----------|----------|
| load1 / N_cpu | baseline / Netdata | **> 0.5** дольше **5 мин** | **> 1.0** дольше **5 мин** | предупреждение → разобрать причину; критично → см. план защиты нагрузки |
| MemAvailable | `free -h` / Netdata | **< ~880 MiB** (в `slo-thresholds.env` как `MEM_AVAIL_WARN_KB`; черновик 974 MiB давал частые ложные предупреждения) | **< 450 MiB** (близко к минимуму ~517 MiB в логе) | критично → проверить утечки, релей, не убивать OOM без анализа |
| RSS xray | Netdata / `ps` | **> 55 MiB** устойчиво (близко к p95/max периода) | **> 80 MiB** | смотреть число соединений, политику buffer, релей |
| est TCP :443 | baseline / `ss` | **> 100** в минутном снимке часто | **> 150** | коррелировать с load; при росте пользователей — масштабирование |

**Гистерезис** (если включите stress-mode по памяти): порог «включить» ≠ «выключить», например критично по RAM **450 MiB** на включение и **600 MiB** на выключение.

## systemd MemoryMax (черновик)

По данным: 99p RSS xray ~**48 MiB**, max в выборке ~**62 MiB**. Разумный старт: **`MemoryMax=384M`** или **`512M`** (запас под всплески). Слишком низкий лимит даст OOM и обрыв всех VPN — не ставьте без графиков под нагрузкой.

Пересмотр порогов: **через 1–2 недели** после накопления данных или при смене числа пользователей / режима routing (весь трафик через релей).

## Как обновить эту таблицу

Повторный разбор лога на сервере:

```bash
# скопируйте вывод в заметку или обновите цифры вручную
awk -F'[= ]' '/mem_avail_kb=/ {print}' /var/log/veil-baseline.log | tail -500
```

Или экспортируйте CSV из Netdata за неделю и скорректируйте p5/p95.

## Установлено на сервере

- **systemd:** `MemoryMax=512M` для `xray.service` (`/etc/systemd/system/xray.service.d/override.conf`).
- **Пороги в конфиге:** [scripts/load-protection/slo-thresholds.env](../../../scripts/load-protection/slo-thresholds.env).
- **Проверка:** [scripts/load-protection/check-slo.sh](../../../scripts/load-protection/check-slo.sh) каждые **5 минут** (cron), лог **`/var/log/veil-slo.log`**, ротация `/etc/logrotate.d/veil-slo`. Код выхода: `0` ok, `1` warn, `2` crit.
