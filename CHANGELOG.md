# Changelog

Все значимые изменения в проекте будут документироваться в этом файле.

## [1.3.10] - 2026-02-24

### 🔧 Исправлено
- Устранены частые таймауты/зависания API под нагрузкой (могли приводить к `504` на Nginx) за счёт сокращения блокирующих операций в обработчиках и фоновых задачах.

### ✅ Изменено
- `POST /api/keys`: создание ключа стало быстрым — операции добавления пользователя в Xray и обновления конфигурации выполняются в фоне, без ожидания в рамках HTTP-запроса.
- Фоновая синхронизация трафика оптимизирована: выполняется батчами и реже (по умолчанию раз в 10 минут), не удерживает DB-сессию во время вызовов Xray.
- SQLite: включён WAL/`busy_timeout`, снижена вероятность блокировок и ошибок пула соединений.

### Технические детали
- Вызовы Xray CLI и операции с конфигом вынесены из event loop в отдельный поток.
- Обновлён unit systemd сервиса: добавлены лимиты concurrency/backlog для защиты от перегруза.

## [1.3.9] - 2026-02-15

### Добавлено
- Параметр `reality_flow` в настройках (по умолчанию `xtls-rprx-vision`) для совместимости с Xray 26.x, в котором flow `"none"` для VLESS не поддерживается.

### Изменено
- Добавление пользователей в Xray (API и конфиг) и генерация VLESS-ссылок используют flow из `settings.reality_flow`.
- Документация: README, CHANGELOG, API_DOCUMENTATION обновлены под Xray 26.x и настраиваемый flow.

### Технические детали
- Xray-core 26.x: в конфиге и при добавлении пользователей используется flow `xtls-rprx-vision` (или значение из `REALITY_FLOW`). Существующий конфиг на сервере обновляется заменой `"flow": "none"` на `"flow": "xtls-rprx-vision"`. Клиентам нужны ссылки с новым flow и поддержка Vision в клиенте.

## [1.3.8] - 2026-02-15

### 🔧 Исправлено
- Сброс трафика: вместо remove/add пользователя в Xray используется `xray api statsquery -reset=true` с pattern по email — счётчики обнуляются в Xray, трафик не «возвращается» после синхронизации
